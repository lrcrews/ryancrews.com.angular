Why Doesn't Six Fear Seven In Base Eight?

Because Seven Ten Eleven.

Let's talk about numbers, hooray!

We're all happy in our day to day lives using base ten, it's nice. We don't even think of it as "base ten" (...or "decimal"), they're just numbers, nothing special.

But what if those numbers were kind of special, and what if we all actually understood that joke up there without even realizing it? We'd be like Rob Schneider in Surf Ninjas with the awesome 'what if' power! ...What if we lose this power?

Intrigued eh? Great, let's do some math! No wait, don't leave!

I promise it's just the easy stuff, no letters or Greek characters are involved. Multiplication and addition will take us to this punchline. But first, let us start with a number we know the value of, one hundred and twenty three, 123, one two three. How does 1|2|3 get it's value?

Well, in base 10 it's due to the following math:

  • value = 3 * 100 + 2 * 101 + 1 * 102
  • value = 3 * 1 + 2 * 10 + 1 * 100
  • value = 3 + 20 + 100
  • value = 123

You see each digit in a number (in any base) is itself times the base (10 in this case) raised to the power of the position. The position goes from the right of the number to the left, and begins counting at 0. This is why we reversed the number up there and you see that 3 is multiplied by the base value 10 raised to its position 0, and the left most number 1 is multiplied by the base value 10 raised to its position 2 (a.k.a. ten squared, a.k.a. ten times ten, a.k.a. rin-ten-ten the super number).

Awesomeness, we now know the math to get the value of a number in base ten... which isn't amazingly useful since we could just, you know, read the number. Let's do that same math then, but this time let's do it in base eight:

  • value = 3 * 80 + 2 * 81 + 1 * 82
  • value = 3 * 1 + 2 * 8 + 1 * 64
  • value = 3 + 16 + 64
  • value = 83

It's a different value, hooray!

And, after these two examples, we notice a couple of things that we learned oh so long ago.

First, any number raised to the power of 0 is 1, giving us the 3 * 1 in both cases above.

Second, any number raised to the power of 1 is itself, so we ended up with 2 * 10 in base 10 and 2 * 8 in base 8.

Alright, I know what you're thinking. You're thinking, "The 50th anniversary X-Men story Battle of the Atom was rather mundane", and you're not wrong, but it doesn't really doesn’t apply to this discussion. Perhaps you were also thinking, “Hey, let’s do it in binary!”. I like the spirit, and in the right context that could be a very sexy phrase, however I fear I must point out that we can’t do it in binary.

Binary, or base two, may only have a 0 or a 1 in any given digit. That makes our example number of 123 invalid. This is the last thing to remember about numbers though! Every digit of any number may only have values from 0 to the base - 1. So, the base 10 numbers we all know and love only have the values 0 to 9, binary 0 to 1, and the popular CSS color codes are hexadecimal, or base 16, and have values from 0 to F for any given digit (where F is 15). Which leaves us with base 8, where you can only count to seven before you have to increment the digit to the left and reset.

So, in base 10, six is afraid of seven because seven eight nine. But, in base 8, six has no fear of seven because seven ten eleven.

This type of thing shows up in technical interviews from time to time as arrays are wonderfully useful structures to calculate any number's value in any base system. You can check out some Objective-C code showing this on github.
